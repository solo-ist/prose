# Issue Documentation

This folder contains documentation for GitHub issues that require detailed design work, implementation notes, or test scenarios.

## Convention

For each issue that needs documentation, create a folder named after the issue number:

```
docs/issues/
├── 42/
│   ├── notes.md         # Development notes and decisions
│   ├── design.md        # Design document or mockups
│   └── test-cases.md    # Test scenarios
├── 123/
│   └── notes.md
```

## When to Create Issue Docs

Create documentation for:
- Features requiring architectural decisions
- Complex bug investigations
- UI/UX design work
- Implementation plans spanning multiple PRs

Simple bug fixes and small features don't need dedicated folders.
